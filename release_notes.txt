Version 4.0.0
================================
This package contains the following extensions:
    - Jumi component (com_jumi)
    - Jumi system plugin (plg_system_jumi)
    - Jumi module (mod_jumi)

Major changes in 4.0.0 (Joomla! 6 migration):
    - Rewritten for the Joomla 4/5/6 extension architecture:
        * Namespaced PSR-4 MVC (base namespace Jumi\...).
        * Dependency-injection service providers (services/provider.php) for the
          component, module and plugin.
        * The module now uses a namespaced Dispatcher + Helper.
        * The system plugin now implements SubscriberInterface and receives its
          dependencies (application, database) via the DI container.
    - Removed all deprecated Joomla APIs (JFactory, JControllerLegacy,
      JModelLegacy, JViewLegacy, jimport, JRequest, JResponse, JObject,
      JArrayHelper, JString, JToolBarHelper, ...).
    - Replaced the legacy root router.php and the separate "Jumi Router" system
      plugin with a component SEF router (com_jumi Site\Service\Router).
    - Distribution is now a Joomla package (pkg_jumi) instead of a component
      that self-installs sub-extensions via a JInstaller script.
    - Database queries use prepared statements / parameter binding throughout.
    - The #__jumi table now uses InnoDB / utf8mb4 and adds checked_out_time for
      proper record check-in/out support.
    - Security hardening:
        * File includes are canonicalised and constrained to the configured base
          directory (or the Joomla root), defeating directory traversal.
        * {jumi} tags are only processed in prepared content items, not in the
          rendered page body, so reflected user input is never executed.
        * The nested tag replacement is bounded to prevent runaway loops, and
          output buffers are always cleaned up when embedded code throws.
        * The Smart Search (com_finder) indexer never executes Jumi applications.
        * The bundled Blogspot demo sanitises remote feed HTML with an
          attribute-aware allow-list filter, validates the blog id, fetches the
          feed with a bounded timeout, parses it with network access disabled
          (LIBXML_NONET) and caches it in the Joomla cache directory.

Installation:
    Install the pkg_jumi package as a standard Joomla! extension. All bundled
    extensions are installed automatically and the system plugin is enabled.

Uninstallation:
    Uninstalling the package removes the component, module and plugin. The
    component uninstall drops the #__jumi table.
